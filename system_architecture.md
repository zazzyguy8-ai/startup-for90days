# System Architecture — save state

## Patterns in use (read before extending)

- **Demo-mode fallback everywhere.** `process.env.OPENAI_API_KEY` absent → `lib/mock.ts` generates deterministic reports (seeded PRNG from idea+audience; keep `Math.abs` in `rng` — `Math.imul` overflows negative). `supabaseConfigured` (lib/supabase.ts) false → `lib/store.ts` falls back to localStorage (`msv_ideas`, `msv_chat_<id>`). Any new feature must work in both modes.
- **Single report schema.** `ValidationReport` in `lib/types.ts` is the contract shared by the OpenAI JSON prompt (`lib/prompts.ts`), the mock generator, all report UI tabs, and the PDF exporter. Change it in all four places at once.
- **framer-motion constraint:** `AnimatePresence` exit animations never complete under this React 19/motion 12 combo (children fade to opacity 0 but never unmount). Do NOT reintroduce `AnimatePresence mode="wait"`; use keyed `motion.div` with enter-only animations (see IdeaClient tab switcher).
- **Next 16 conventions:** `params` is a `Promise` — server pages `await params` and delegate to a `"use client"` component (see `app/idea/[id]/page.tsx`).
- **Design system** lives in `app/globals.css` as CSS variables + utility classes (`.glass`, `.glass-strong`, `.btn-primary`, `.btn-ghost`, `.input-glass`, `.chip`, `.text-gradient`, aurora). Dark is default (inline script in layout.tsx sets `.dark` before hydration; stored in `localStorage.msv_theme`). Use the variables, not hardcoded colors.
- **Score color scale** is centralized in `scoreColor()` (components/ScoreRing.tsx): ≥75 green / ≥55 indigo / ≥38 amber / else red — mirrors verdict labels.
- Dev server for previews runs via top-level `~/.claude/launch.json` entry `micro-saas-validator` on port 3010 (`npm --prefix … run dev -- -p 3010`).
- **Build My Startup** (`/idea/[id]/build`): `BuildKit` lives on `SavedIdea.build` (Supabase `ideas.build jsonb` — existing installs need `alter table public.ideas add column if not exists build jsonb`). Generation is **per-section** via POST `/api/build` `{section, input, report}` — six `BuildSectionId`s generated sequentially client-side (BuildClient.generateAll) so each OpenAI response stays small; array sections (tasks/code/prompts) are unwrapped from JSON-mode object wrappers in the route. Demo fallbacks in `lib/build-mock.ts`. Task-done state and regenerated sections persist through `saveBuild()` → `saveIdea()`.
- **Full app generation** (`lib/project-generator.ts`): the AI designs a compact `ProjectSpec` (app name, input fields, the product's core prompt with `{{fieldId}}` placeholders, result sections, accent color) via `/api/build {section:"project"}`; `generateProjectFiles(spec, report)` then deterministically materializes a complete 19-file Next.js 15 app (landing/pricing/core workflow/dashboard, OpenAI-or-demo `/api/generate`, Stripe `/api/checkout`). This split keeps the AI response tiny while the codebase is guaranteed to compile — VERIFIED by building a generated project (`npm install && npm run build && npm start` + curl). Files regenerate from the spec, and download is client-side via dynamic `import("jszip")`. Stored as `BuildKit.project {spec, files}` (optional — older kits show a generate CTA).
- **Pacing + verification pattern:** long-feeling AI work is intentional UX. `/api/validate` runs a two-pass generate→verify chain when a key is present (`VERIFY_SYSTEM_PROMPT` in lib/prompts.ts, disable via `OPENAI_VERIFY=0`). Client-side, the validate page and BuildClient run the fetch **in parallel** with a scripted activity sequence (`Promise.all([request, sequence])`) — never navigate/finish before both resolve. Activity lines are personalized from the user's input; keep this pattern for any new AI feature so demo mode and real mode feel identical.
- **Startup Builder** (step 3 view, FullProjectView): agentic loop — instruction → POST `/api/build {section:"revise", spec, instruction}` → updated `ProjectSpec` + `changes[]` (OpenAI via `buildRevisePrompt`; demo fallback `mockReviseSpec` parses names/colors/taglines incl. Slovak words — regexes must stop at ` and/a/aby `, see the lookahead patterns) → `generateProjectFiles` → file-by-file write animation → `onSave` persist → iframe preview re-render. The live preview is a self-contained HTML doc from `lib/preview-generator.ts` (`generatePreviewHtml(spec, report)`, sandboxed `srcDoc`, inline JS demo) — update it whenever the generated app's look changes.
- lucide-react v1 in this project has **no brand icons** (Twitter, Linkedin, etc. removed) — check `node -e "require('lucide-react').X"` before using a new icon.
